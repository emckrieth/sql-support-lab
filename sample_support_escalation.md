# Support Query Guide

## Why these queries matter
These queries mirror common support tasks:
- confirm whether failures are isolated or repeated
- detect duplicate customer records
- identify users missing a completed verification
- summarize repeated incident signatures
- distinguish application issues from bad or missing data

## Example support workflow
1. Confirm the user and timestamp.
2. Query related verification records.
3. Check whether the webhook was delivered.
4. Compare the error pattern against known incidents.
5. Escalate with query results attached.
